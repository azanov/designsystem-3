Action Buttons

Button Styles and States

There are 4 button styles, and 4 states of each style. Their style names correspomd to the CSS classes used by Bootstrap.

All buttons share the base CSS class .btn. The other types are created by adding additional classes below, as in class="btn btn-primary".

Styles:
  • .btn-primary: The button is intended to be the default action on the screen. There should be only one default button in a group of buttons.
  • .btn-default: A normal button.
  • .btn-link: When you want an action to be styled like a hyperlink, but need it properly aligned when grouped with the buttons above.
  • .btn-pill: A "pill-shaped" button with completely rounded side caps. This class is to be added to one of the previous three, to add the pill shape.
States
  • Normal
  • Hovered
  • Focused
  • Disabled
Currently please use Pill Buttons only for landing or marketing pages

Sizes

Buttons are provide in 4 sizes: the default size, which does not require a class; .btn-lg, .btn-sm, and .btn-xs.

Icons and Badges

Icons or badges can be added to button labels.

Block

By adding the class .btn-block to any button, it will take up the maximum width of its container.

Group

Group a series of buttons together on a single line by wrapping with div class="btn-group" to create a button group.

You can also add/use icons to create a quick toolbar. For guidelines on Toolbars, please see below and Elements > Tables > Datatables.

Toolbar Buttons

Wrap ordinary buttons or icon buttons in a div class="toolbar" to style them as toolbar buttons. These are used over a table, for example. Notice they are slightly smaller and their outlines are dark grey.

Toggle

Toggle buttons are buttons that have a "on" and an "off" state. They can be grouped to create controls for multiple states. Grouped toggle buttons can be act like either "checkboxes" or "radio buttons," to allow for multiple active or mutually exclusive buttons, respectively.

For more info on toggle buttons and their methods, please see the Bootstrap documentation.

Angular JS developers might also want to consider Angular Bootstrap components which provide easier inegration with apps.

Single Dropdown

Turn a button into a dropdown toggle with some basic markup changes.

Split Dropdown

Place a .btn-group within another .btn-group when you want dropdown menus mixed with a series of buttons.

Writing Button Copy

All buttons should:

  • Use title capitalization
  • Never conclude with a period
  • Capitalize all words containing four or more letters
Length and Wording

Keep word counts to three words or fewer if possible. If you need to use more than three words, use the smallest number of words for conveying the button’s call to action.

Use action verbs (“get,” “see,” “filter,” “send,” “download,” “go to,” “share,” etc.) and make sure that the terminology you’re using is customer-facing and consistent with the rest of your product.

Use “OK” in button labels, not “Okay” or “okay”.

See also: writing

Links

Link Style

You can include an icon with your link if it will increase the user’s understanding. But avoid using icons that are unclear or unnecessary.

Writing Link Copy

Hyperlinked buttons should:

  • Feel conversational
  • Use active writing and be positive
  • Use sentence capitalization
  • Never conclude with punctuation unless they are formed as questions

See also: writing

Tiles

Usage

We use tiled buttons to highlight functionality that defines the purpose of a particular product. These buttons:

  • Can be either bordered or borderless
  • Are visually prominent and highly graphical
  • Make use of icons or recognizable brand elements
  • Are designed to be easy for users to see and click on

Breadcrumbs

Usage

Breadcrumb navigation should:

  • Reflect a product’s static page hierarchy
  • Appear above the H1 header page titles

Style

Breadcrumb navigation:

  • Uses title capitalization
  • Can treat the “Home” level as optional
  • Uses forward slashes (not chevrons, bullets, or colons) as dividers
  • Uses an “em” space on either side of forward slash dividers
  • Displays the current page location, which is not hyperlinked
  • Can abbreviate longer page titles if it’s possible to do so without sacrificing clarity
