Overview

PB-Animate has been built to handle different kinds of animations. Most people will be primarily focussed on entrance/exit animations. These animations do a great deal to smooth out the overall fluidity of an application. Emphasis animations should be less commonly used. These come into play when you think a user might be stuck on what step comes next, or need to guide their eye to something in your UI.

Coding with PB-Animate

Design System's custom animations are built for Angular projects. In order to use these animations, make sure you have the ng-animate module included in your app. Any additional animations applied with PB-Animate should not interfere with a non-angular animations.

All of the PB-Animate animations have been built with CSS3 Keyframes.

Compatable Angular Attributes

The following angular directives will work with PB-Animate classes.

  • ng-if
  • ng-show
  • ng-hide
  • ng-repeat
  • ng-class (coming soon)

There are no restrictions on how these data attributes are added in the HTML template. If you would prefer to prefix your attributes with data- that will not interfere with the animations.

Crafting Animations with PB-Animate

The pb-animate classes can be applied in two ways. The simplest is to add PB-Animate classes directly to the HTML element. This is useful for rapid prototyping. However, it is recommended that once you are satisfied with your final product that you move your classes into an @extend in Sass so your templates are easier to manage.
